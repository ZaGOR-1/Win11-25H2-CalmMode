# ROADMAP — Win11 25H2 CalmMode

План розвитку за пріоритетом. Кожен пункт лишається в рамках філософії проєкту: **безпека,
прозорість, відкатність, opt-in для ризикованого, без зовнішніх залежностей, сумісність із
Windows PowerShell 5.1**.

Легенда ризику: 🟢 низький · 🟡 середній · 🔴 потребує обережності/дизайну.

> 5C — у робочому дереві (не закомічено); `AUDIT.md` описує v2.5.

## Фаза 6 — далі (нові напрями)

Фазу 5 завершено; це — кандидати на подальший розвиток, згруповані за темами.

### 6A. Довіра до Apply-шляху (найбільша реальна прогалина) 🟡

- [ ] **6.1 Прогін у VM і фіксація результатів.** За всю розробку `Apply`, `-RestoreFrom` (реальний `reg import`), `-EnableSystemProtection` і GUI «Undo last Apply» перевірялись лише **статично + error-шляхи**. Потрібен документований прогін у чистій VM: `Audit` (без/з адміном) → `Apply` → reboot → `Verify` → `-RestoreFrom` → повторний `Verify`. Результати (скріншоти/звіти) — у `AUDIT.md` або окремий `docs/vm-validation.md`.
- [ ] **6.2 Тест ідемпотентності.** Повторний `Apply` поспіль не має давати `Changed` вдруге (усе вже `Compliant`/`AlreadyConfigured`). Зараз це припущення, не перевірене автоматично — додати сценарій (у VM або через мок реєстру).

### 6B. Зручність і доступність 🟢

- [ ] **6.3 Готові пресети конфігів.** Кілька прикладних JSON (`presets/minimal.json`, `balanced.json`, `strict.json`) у форматі `-ConfigPath`; у GUI — список «Load preset». Дешево, помітна цінність, нуль ризику.
- [ ] **6.4 Локалізація GUI (UA/EN).** Інтерфейс зараз лише англійською, а користувач україномовний. Простий словник рядків + автовибір за культурою або перемикач. 🟡 (тримати тонким шаром).
- [ ] **6.5 Порівняння двох звітів.** `-CompareReports <before.json> <after.json>` → що змінилось між прогонами (статуси/значення). Корисно «до/після Apply» і для регресій. 🟢

### 6C. Контент — нові політики 🟡

- [ ] **6.6 Ревізія політик під свіжі UBR 25H2.** Періодично перевіряти нові «calm»-релевантні політики (lock-screen Spotlight, Start «Phone Link»/recommendations, нові Recall/Copilot/Click-to-Do ключі). Кожна — суворо за дисципліною `add-policy`: точний Path/Name/Type, `Confidence`, `MinBuild`/`MinUBR`, `Editions`, чесний статус. Нічого не подавати як «гарантовано працює».

### 6D. Інструментарій і гігієна 🟢

- [ ] **6.9 (опц.) Read-only перевірка нової версії.** Порівняти локальний `VERSION` з останнім тегом релізу й **повідомити** (без автозавантаження/виконання коду — це проти філософії). Додає мережеву залежність → лише opt-in прапором. 🟡

## Non-goals (свідомо НЕ робимо)

- **Рефакторинг рушія на модулі.** 1851 рядок — помітний моноліт, але він робочий, покритий тестами і має лишатися одним самодостатнім `.ps1` для простоти роздачі. Розбивка = ризик без користувацької цінності (CLAUDE.md: «не роби rewrite без потреби»). Робити лише якщо зявиться конкретна потреба.
- **Компільований `.exe`-лаунчер / base64 / завантаження коду** — проти філософії (AV/SmartScreen, прозорість).
- Будь-що із забороненого списку (див. нижче).
